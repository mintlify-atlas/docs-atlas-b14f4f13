## Tu Primera Venta

Esta guía te llevará desde la instalación hasta completar tu primera venta en menos de 10 minutos.

<Steps>
  <Step title="Instalar el sistema">
    Instala POS CDP Lite usando tu método preferido:

    <CodeGroup>
      ```bash Composer
      composer create-project mroblesdev/pos-cdp-lite
      cd pos-cdp-lite
      ```

      ```bash Git
      git clone https://github.com/mroblesdev/pos-cdp-lite.git
      cd pos-cdp-lite
      ```
    </CodeGroup>
  </Step>

  <Step title="Configurar base de datos">
    Crea la base de datos y configura las credenciales:

    ```sql
    CREATE DATABASE IF NOT EXISTS `pos-cdp-lite` 
    DEFAULT CHARACTER SET utf8mb4 
    COLLATE utf8mb4_unicode_ci;
    ```

    Copia y edita el archivo de configuración:
    ```bash
    cp env .env
    ```

    Edita `.env` con tus credenciales:
    ```bash
    database.default.database = pos-cdp-lite
    database.default.username = root
    database.default.password = tu_password
    ```
  </Step>

  <Step title="Inicializar datos">
    Ejecuta las migraciones y seeders para crear las tablas:

    ```bash
    php spark migrate
    php spark db:seed CargaSeeder
    ```

    Esto creará:
    - Tablas de usuarios, productos, ventas y configuración
    - Usuario administrador por defecto
    - Datos de ejemplo (opcional)
  </Step>

  <Step title="Iniciar el servidor">
    Levanta el servidor de desarrollo:

    ```bash
    php spark serve
    ```

    <Info>
      El sistema estará disponible en `http://localhost:8080`
    </Info>
  </Step>

  <Step title="Iniciar sesión">
    Abre tu navegador y accede a `http://localhost:8080`

    Inicia sesión con las credenciales por defecto:
    - **Usuario:** `admin`
    - **Contraseña:** `admin`

    <Warning>
      Cambia la contraseña después del primer inicio de sesión desde el menú de usuario.
    </Warning>
  </Step>

  <Step title="Agregar un producto">
    1. Ve a **Productos** en el menú lateral
    2. Haz clic en **Nuevo Producto**
    3. Completa el formulario:
       - **Código:** 001
       - **Nombre:** Producto de Prueba
       - **Precio:** 100.00
       - **Inventariable:** Sí
       - **Existencia:** 10
    4. Haz clic en **Guardar**

    <Tip>
      Los productos inventariables deducirán automáticamente su existencia al vender.
    </Tip>
  </Step>

  <Step title="Realizar una venta">
    1. Ve a **Caja** en el menú
    2. Escanea o escribe el código del producto (001)
    3. El producto se agregará al carrito
    4. Haz clic en **Cobrar**
    5. Se generará automáticamente un ticket PDF

    <Info>
      Puedes agregar múltiples productos y ajustar cantidades antes de cobrar.
    </Info>
  </Step>

  <Step title="Ver el historial">
    1. Ve a **Ventas** para ver todas las ventas realizadas
    2. Puedes:
       - Reimprimir tickets
       - Cancelar ventas (se restaura el inventario)
       - Ver detalles de cada venta
  </Step>
</Steps>

## Funcionalidades Clave

### Dashboard

- Resumen de ventas del día
- Acceso rápido a funciones principales
- Estadísticas básicas

### Caja de Cobro (POS)

La interfaz principal para ventas incluye:

- **Búsqueda rápida:** Por código o nombre de producto
- **Carrito temporal:** Agrega múltiples productos antes de cobrar
- **Ajuste de cantidad:** Aumenta o disminuye cantidades
- **Total en tiempo real:** El total se actualiza automáticamente
- **Generación de ticket:** PDF automático al completar la venta

```php Ejemplo de flujo de venta
// 1. Usuario escanea código o busca producto
// 2. Sistema valida existencia (si es inventariable)
// 3. Producto se agrega a temporal_caja con id_venta temporal
// 4. Al cobrar, se crea registro en ventas
// 5. Se generan detalles de venta
// 6. Se actualiza inventario
// 7. Se limpia temporal_caja
// 8. Se genera ticket PDF
```

### Historial de Ventas

<Accordion title="Consultar Ventas">
  - Ve todas las ventas activas
  - Filtra por fecha, folio o cliente
  - Accede a detalles completos de cada venta
</Accordion>

<Accordion title="Reimprimir Tickets">
  - Regenera el PDF de cualquier venta
  - Útil para duplicados o pérdidas
</Accordion>

<Accordion title="Cancelar Ventas">
  - Cancela ventas incorrectas
  - El inventario se restaura automáticamente
  - Las ventas canceladas se marcan pero no se eliminan
</Accordion>

### Reportes

<CardGroup cols={2}>
  <Card title="Reporte de Ventas" icon="chart-line">
    Ventas por rango de fechas con totales y cantidad de transacciones
  </Card>
  <Card title="Reporte de Productos" icon="box">
    Inventario actual con precios y existencias
  </Card>
</CardGroup>

Ambos reportes se generan en PDF para fácil impresión y archivo.

### Configuración del Sistema

Personaliza la información de tu tienda:

- **Nombre de la tienda:** Aparece en tickets y reportes
- **Dirección:** Se imprime en los tickets
- **Teléfono:** Contacto para clientes
- **Leyenda del ticket:** Mensaje personalizado (ej: "Gracias por su compra")
- **Folio de ventas:** Controla la numeración de ventas

## Atajos y Tips

<Tip>
  **Búsqueda rápida en Caja:** En la interfaz de caja, puedes usar el autocompletado escribiendo parte del código o nombre del producto.
</Tip>

<Tip>
  **Productos sin inventario:** Si vendes servicios, marca el producto como "No inventariable" para evitar control de existencias.
</Tip>

<Tip>
  **Cambiar contraseña:** Ve al menú de usuario → Cambiar Contraseña para actualizar tus credenciales.
</Tip>
